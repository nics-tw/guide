政府網站設計原則親和力宣告

這是一份由 PDIS 發表的親和力宣告。並於 2024 年 3 月 27 日轉由 NICS 接續維護。

支持親和力的舉措

PDIS 採用下列舉措，以確保政府網站設計原則具備親和力：

  • 在我們的任務宣告裡納入親和力。
  • 在我們內部政策的各個環節均考量親和力。
  • 諮詢親和力總顧問。

符合狀態

全球資訊網協會在 Web Content Accessibility Guidelines (WCAG) 定義網頁設計師及開發者應該如何促進身心障礙者使用網頁的親和力，在其中對親和力的符合狀態分成三個符合等級：等級 A、等級 AA、等級 AAA。政府網站設計原則 完全符合 WCAG 2.1 等級 AA 的要求。「完全符合」的意思是所有內容完全依照親和力標準的要求來製作，沒有任何例外。

額外的親和力考量

網站無障礙規範(110.07)

回饋管道

我們歡迎任何對政府網站設計原則的親和力相關回饋。如果您在政府網站設計原則遭遇任何使用障礙，請透過下列方式讓我們知道這個情況：

  • 電子郵件： ikngoo@nics.nat.gov.tw

我們會盡量在兩個工作天內對您的回饋內容提出回應。

技術規格

政府網站設計原則的親和力設計係依賴下列技術實現，使您在各種網頁瀏覽器及輔助科技的搭配上均可使用：

  • HTML
  • WAI-ARIA
  • CSS
  • JavaScript

對於親和力標準的符合性係依賴上述這些技術才得以實現。
